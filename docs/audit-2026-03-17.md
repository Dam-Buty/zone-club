## 1. Performance

### CRITIQUE

#### P-01. N+1 Query: enrichRental() dans getUserActiveRentals()
**Fichier** : `lib/rentals.ts:96-104`

`getUserActiveRentals()` execute 1 query rentals + N queries films (1 par location active). Appele sur chaque `/api/me` (chargement app + regulierement pendant la lecture).

**Impact** : 5 locations = 6 queries SQL. Route `/api/me` ralentie de 50-100ms.

**Correctif** : JOIN unique :
```sql
SELECT r.*, f.* FROM rentals r
JOIN films f ON r.film_id = f.id
WHERE r.user_id = ? AND r.is_active = 1 AND r.expires_at > datetime('now')
```

#### P-02. Chat: getFilmsByAisle() appele 10+ fois par message
**Fichier** : `lib/chat.ts:23-34, 109-120`

`buildSystemPrompt()` boucle sur 10 allees avec une query DB par iteration. Execute a chaque message chat.

**Impact** : +100-200ms de latence par message chat.

**Correctif** : Query unique `SELECT aisle, * FROM films WHERE is_available = 1` puis grouper en memoire.

#### P-03. enrichFilmsWithStock() scanne toute la table rentals par allee
**Fichier** : `app/api/films/aisle/[aisle]/route.ts:11-23`

13 allees chargees en parallele au demarrage = 13 full table scans sur `rentals`.

**Impact** : Temps de chargement initial proportionnel a `O(nombre_locations)`.

**Correctif** : Index sur `rentals(film_id, is_active, expires_at)`.

### HAUTE

#### P-04. Index manquants sur colonnes tres requetees
**Fichier** : `lib/schema.sql`

Indexes manquants :
```sql
CREATE INDEX IF NOT EXISTS idx_rentals_film_id_active ON rentals(film_id, is_active, expires_at);
CREATE INDEX IF NOT EXISTS idx_rentals_user_id_active ON rentals(user_id, is_active);
CREATE INDEX IF NOT EXISTS idx_rentals_expires_at ON rentals(expires_at);
```

**Impact** : Full table scan sur chaque requete location/film.

#### P-05. VHSPlayer: 15+ variables d'etat sans memoisation
**Fichier** : `src/components/player/VHSPlayer.tsx:32-100`

Chaque changement d'etat declenche un re-render du composant entier (600+ lignes). Le scrubbing rapide (`currentTime`) peut generer 30+ re-renders/sec.

**Impact** : Saccades sur appareils bas de gamme pendant les interactions player.

#### P-06. GPU compute inconditionnel sur 1002 instances
**Fichier** : `src/components/interior/CassetteInstances.tsx:160-190`

Le compute shader tourne chaque frame pour 1002 instances, meme quand aucune cassette n'est ciblee.

**Impact** : ~5-10ms GPU gaspilles en idle.

**Correctif** : Dirty flag — ne lancer le compute que quand une cassette est ciblee ou change d'etat.

#### P-07. Raycaster: 526 objets testes 30 fois/sec
**Fichier** : `src/components/interior/Controls.tsx:200-250`

Malgre les layers (1+2), 526 objets sont raycasted 30x/sec = ~15780 tests/sec.

**Impact** : ~3-5% CPU desktop en idle.

#### P-08. Prefetch posters: 500+ requetes images non annulables
**Fichier** : `src/App.tsx:58-81`

Module-level Promise.all qui prefetch tous les posters. Sur 3G, cela genere 500+ requetes images jamais annulees.

**Impact** : 5-20MB de bande passante gaspillee sur reseau lent.

### MOYENNE

#### P-09. Service Worker: pas de versioning automatique des assets
**Fichier** : `public/sw.js`

VERSION manuel (`v2`). Si oubli de bump, les utilisateurs gardent des assets perimees.

#### P-10. Pas de lazy loading pour les modals lourds
**Fichier** : `src/App.tsx:378-386`

AuthModal, ReviewModal, BoardOverlay montes en permanence. +200-400KB JS parse au chargement.

#### P-11. Textures canvas recreees par instance
**Fichier** : `src/components/interior/Aisle.tsx:125-167`

`createPrivateSignTexture()` appele 3 fois (1 par instance) au lieu de partager 1 texture.

#### P-12. Cast hook: requetes inflight jamais annulees
**Fichier** : `src/components/player/VHSPlayer.tsx:102-120`

Les refs cast (`castFilmIdRef`, `castDurationRef`, `lastKnownCastTimeRef`) ne sont jamais nettoyees au unmount.

## 2. Securite

### CRITIQUE

#### S-01. Fallback HMAC secret en dur (CWE-798)
**Fichier** : `lib/session.ts:4`
```typescript
const SECRET = process.env.HMAC_SECRET || 'dev-secret-change-in-production';
```

Si `HMAC_SECRET` absent en production, les tokens de session peuvent etre forges avec le secret connu.

**Scenario d'attaque** : Forgerie de tokens → usurpation d'identite (y compris admin).

**Correctif** : Supprimer le fallback, `throw new Error()` si variable absente en production.

#### S-02. API key: user ID accepte depuis le header client (CWE-639)
**Fichier** : `lib/session.ts:41-48`
```typescript
const userId = parseInt(req.headers.get('x-user-id') || '1', 10);
```

Apres verification de l'API key, le code fait confiance au header `x-user-id` du client. Quiconque possede l'API key peut usurper n'importe quel utilisateur.

**Correctif** : Mapper l'API key a un user ID fixe cote serveur, ne pas accepter `x-user-id` du client.

#### S-03. Acces admin sans defense en profondeur (CWE-639)
**Fichier** : Routes `app/api/admin/`

Les endpoints admin verifient `!user?.is_admin` depuis la session, mais combines avec S-01 et S-02, un secret HMAC connu ou une API key volees = acces admin complet.

**Correctif** : Rate limiting sur auth, audit logging admin, tokens a duree courte.

### HAUTE

#### S-04. Symlinks sans validation de chemin (CWE-59)
**Fichier** : `lib/symlinks.ts:36-47`

`filePaths.vf/vo` proviennent de la DB (via Radarr) sans validation de traversal (`../`).

**Correctif** : `path.resolve()` puis verifier que le chemin commence par le repertoire media attendu.

#### S-05. Pas de CSRF protection (CWE-352)
**Fichier** : Tous les endpoints POST/PATCH/DELETE

Aucun token CSRF. Attenuation partielle par `SameSite=Strict`, mais insuffisant si l'attaque vient du meme site.

**Correctif** : Ajouter un header custom (ex: `X-Requested-With`) et le verifier cote serveur.

#### S-06. Passphrase de recuperation brute-forcable (CWE-330)
**Fichier** : `lib/passphrase.ts`

3 mots × ~100 mots par dictionnaire = ~20 bits d'entropie. Meme avec bcrypt, 1M de combinaisons = brute-force en minutes sur GPU.

**Correctif** : 4-5 mots au lieu de 3, ou rate limiter `/api/auth/recover` (3 tentatives/h/username).

#### S-07. Pas de rate limiting sur les endpoints auth
**Fichier** : `app/api/auth/login/`, `register/`, `recover/`

Brute-force de mots de passe et enumeration de pseudos possibles sans limite.

#### S-08. SSRF potentiel via Radarr (CWE-918)
**Fichier** : `lib/radarr.ts:29-46`

Pas de timeout sur les appels fetch Radarr, pas de whitelist d'endpoints.

**Correctif** : `AbortSignal.timeout(5000)`, whitelist des endpoints Radarr autorises.

#### S-09. Pas de validation d'input sur les reviews (CWE-80)
**Fichier** : `app/api/reviews/[filmId]/route.ts:44`

Pas de validation de longueur max sur `content`. DoS potentiel (review de 10MB).

### MOYENNE

#### S-10. Mot de passe: seule la longueur est verifiee (CWE-521)
**Fichier** : `app/api/auth/register/route.ts:17-18`

Minimum 8 caracteres, pas de complexite requise. `aaaaaaaa` accepte.

#### S-11. Messages d'erreur trop detailles (CWE-209)
**Fichier** : Gestionnaires catch dans les routes API

`(err as Error).message` renvoye au client → fuite d'information.

#### S-12. Session 7 jours sans revocation (CWE-613)
**Fichier** : `lib/session.ts:10-11`

Pas de mecanisme de logout/revocation. Si appareil vole, 7 jours d'acces.

#### S-13. IDOR sur les films non disponibles (CWE-639)
**Fichier** : `app/api/films/[tmdbId]/route.ts`

`getFilmByTmdbId()` ne filtre pas `is_available`. Films caches accessibles par TMDB ID.

#### S-14. Fallbacks env vars a string vide (CWE-798)
**Fichier** : `lib/radarr.ts:118-125`, `lib/symlinks.ts:5-7`

`process.env.RADARR_VO_API_KEY || ''` — echec silencieux au lieu de crash explicite.

#### S-15. Pas de headers de securite HTTP
**Fichier** : `next.config.ts`

Manquants : CSP, HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy.

## 3. Ergonomie / UX

### CRITIQUE

#### U-01. URL localhost en dur dans la recuperation de mot de passe
**Fichier** : `src/components/ui/AuthModal.tsx:84`
```typescript
const response = await fetch('http://localhost:5179/api/auth/recover', {
```

En production, la recuperation de compte est **completement cassee**.

**Correctif** : Utiliser `/api/auth/recover` (URL relative).

### HAUTE

#### U-02. Pas de feedback inline sur le mismatch mot de passe
**Fichier** : `src/components/ui/AuthModal.tsx:65-67`

Le bouton submit se desactive silencieusement. L'utilisateur ne sait pas pourquoi.

#### U-03. Labels de formulaire non associes aux inputs
**Fichier** : `src/components/ui/AuthModal.tsx`

`<label>` sans `htmlFor`, `<input>` sans `id`. Inaccessible aux lecteurs d'ecran.

**Fichier** : `src/components/interior/Controls.tsx`

#### U-05. Player video: aucun raccourci clavier
**Fichier** : `src/components/player/VHSPlayer.tsx`

Pas de space (play/pause), fleches (seek), M (mute), V (piste audio). Desktop keyboard-only inutilisable.

#### U-06. Boutons "S'INSTALLER" < 44px sur mobile
**Fichier** : `src/components/player/VHSPlayer.tsx` (mobileRemotePrimaryBtn)

Certains boutons en `padding: 8px 10px` = ~22px de hauteur. Standard WCAG: minimum 44x44px.

#### U-10. Terminal 450px fixe deborde sur mobile < 480px

`width: 450px` non responsive. **Correctif** : `width: min(450px, 90vw)`.

## 4. Compatibilite navigateurs / plateformes

### CRITIQUE

#### C-01. Aucun fallback WebGL si WebGPU absent
**Fichier** : `src/App.tsx:173-174`

```typescript
if (!navigator.gpu) {
  return <WebGPUNotSupported />;
}
```

Application entierement inutilisable sans WebGPU. Firefox (flag requis), Safari < 17, anciens navigateurs → ecran d'erreur.

**Impact** : ~15-20% des utilisateurs potentiels exclus.

**Correctif** : Fallback WebGL ou instructions claires avec liens vers les guides d'activation.

### HAUTE

#### C-02. Fullscreen API sans prefixes vendor
**Fichier** : `src/components/player/VHSPlayer.tsx:598-601`

Manque `webkitRequestFullscreen()` (Safari), `mozRequestFullScreen()` (ancien Firefox).

**Impact** : Touche F silencieusement inactive sur Safari et anciens navigateurs.

#### C-03. Google Cast SDK sans feedback d'erreur
**Fichier** : `src/hooks/useGoogleCast.ts:149-196`

SDK charge depuis CDN Google avec timeout 8s. Si bloque (proxy, offline), echec silencieux. Cast est Chrome-only — pas de feedback pour Firefox/Safari.

#### C-04. Bouton AirPlay affiche sur navigateurs non-Safari
**Fichier** : `src/components/player/VHSPlayer.tsx:635-666`

`webkitShowPlaybackTargetPicker` est WebKit-only. Bouton visible mais non fonctionnel sur Android/Firefox.

#### C-05. Pointer Lock sans prefixes vendor
**Fichier** : `src/components/interior/Controls.tsx:1120`

Manque `mozRequestPointerLock()`, `webkitRequestPointerLock()`. Controles FPS casses sur Firefox < 50.

#### C-06. Container queries CSS sans fallback
**Fichier** : `src/components/exterior/ExteriorView.module.css`

`container-type: inline-size` supporte Chrome 105+, Safari 16+, Firefox 110+ seulement.

### MOYENNE

#### C-07. Hotkeys basees sur `event.key` au lieu de `event.code`
**Fichier** : `src/components/player/VHSPlayer.tsx:596-627`

Claviers AZERTY/QWERTZ : F, V, et autres raccourcis ne correspondent pas aux memes touches physiques.

**Correctif** : Utiliser `event.code` (ex: `KeyF`) au lieu de `event.key`.

#### C-08. `crypto.randomUUID()` sans polyfill
**Fichier** : `src/components/manager/ManagerChat.tsx`

Absent sur Chrome < 92, Safari < 15.1. Le composant chat crashera.

#### C-09. iOS Safari: bug 100vh
**Fichier** : `app/layout.tsx:17`

`100vh` inclut la barre d'adresse sur iOS Safari. Elements UI disparaissent/reapparaissent.

**Correctif** : Utiliser `100svh` (small viewport height) ou fallback JS.

#### C-10. Video: format unique MP4/H.264, pas de streaming adaptatif
**Fichier** : `src/components/player/VHSPlayer.tsx:1153`

Pas de source WebM/VP9 alternative. Pas de HLS/DASH pour le debit adaptatif.

#### C-11. Service Worker: version manuelle, pas d'auto-invalidation
**Fichier** : `public/sw.js:3`

`VERSION = 'v2'` en dur. Oubli de bump = assets perimees pour tous les utilisateurs.

### BASSE

#### C-12. Pas de polyfill IntersectionObserver (IE11)
#### C-13. DPR clamp 1.5/1.7 = flou sur ecrans haute densite (Galaxy S23+, iPhone Pro)
#### C-14. PWA install prompt non fiable sur Firefox Android
#### C-15. Pas de retry logic sur les appels fetch
#### C-16. localStorage indisponible en navigation privee iOS Safari
#### C-17. Pas de message explicite de blocage IE11
